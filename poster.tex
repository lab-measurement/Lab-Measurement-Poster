\item

in your looong C program?
\item
Use a text processing language to manage your measurement! Use Perl!

\heading{Currently supported hardware}

\vspace*{-3mm}
\placefigure{file=images/hardware}
Hardware driver backends: 
\begin{itemize}
\item
LinuxGPIB and all hardware supported by it
\item
Linux USB-TMC kernel driver
 \item 
NI-VISA (Windows and Linux) and all hardware supported by it
 \item
Oxford Instruments IsoBus
 \item
Raw TCP connection, generic network socket
 \item
RS232 serial port
\end{itemize}

\heading{Internal architecture}

\vspace*{-3mm}
\placefigure{file=images/structure}

\begin{itemize}
\item
Modular structure. Easy to extend with new instrument drivers and connection
types.
\item
Abstract IO-layer makes instrument drivers independent of hardware backends.
\end{itemize}
\vspace*{0.9cm}
\heading{Recent additions}
\vspace*{-3mm}
\begin{itemize}
\item New drivers for R\&S FSV Spectrum analyzer and the R\&S ZVA and ZVM vector
  network analyzers.
\item automated testing of instrument drivers and high-level functionality via
  mocked connection objects. Every commit is checked with continuous
  integration testing on both Linux (Travis CI) and Windows (Appveyor).
\item 
\end{itemize}
